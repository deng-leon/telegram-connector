# Camunda Marketplace Submission Checklist

## Step 1 — Repository readiness

- [ ] Public GitHub repository exists (open source listing) or listing-only strategy confirmed
- [ ] `README.md` explains setup, usage, and configuration
- [ ] `LICENSE.md` exists
- [ ] `LICENSE.md` contains Camunda-required legal disclaimer block before license terms
- [ ] Connector source code and templates are committed

## Step 2 — Submission form readiness

### Contributor information
- [ ] Company / Contributor Name
- [ ] Contact Email (internal only)
- [ ] Website URL (public profile target)

### Connector listing content
- [ ] Marketplace Profile Title
- [ ] Splash Title
- [ ] Five-Word Description
- [ ] Full Description
- [ ] SEO Description (~2 sentences / ~250 chars)
- [ ] Connector Type selected (Outbound / Inbound / both)
- [ ] Application Category selected
- [ ] Industry selected
- [ ] Use Case selected

### Media assets
- [ ] App Listing Logo URL (downloadable, min 512x512, no SVG)
- [ ] App Profile Logo URL (downloadable, min 230x230, no SVG)
- [ ] Screenshot URL(s) (downloadable, recommended min 1500x1200)
- [ ] Promo Video URL (optional but recommended)

### Feature listing
- [ ] Feature Description section completed
- [ ] Feature 1..6 titles and descriptions (as applicable)

### Technical information
- [ ] Runtime model selected (Camunda protocol runtime reuse: yes/no)
- [ ] Connector SDK version
- [ ] Compatible Camunda version(s)

### Repository links
- [ ] GitHub repository URL
- [ ] Outbound Template raw URL
- [ ] Start Inbound Template raw URL
- [ ] Intermediate Inbound Template raw URL
- [ ] Boundary Inbound Template raw URL

### Support links
- [ ] Documentation URL
- [ ] Support URL
- [ ] Video URL (resource tab)
- [ ] Additional Resource URL (PDF, optional)

## Step 3 — Camunda review

- [ ] Submission reviewed for completeness
- [ ] Approval or feedback handled via provided email
